# Contribution policy

qslib does not currently accept unsolicited external contributions. Pull
requests and contribution proposals may be closed without review. Opening the
project requires a later owner decision, a private conduct-reporting channel,
and an adopted code of conduct.

The project is developed under a physics-first, specification-driven process.
Supported changes must preserve the conventions in
[`conventions.md`](conventions.md), use an independent test oracle, and follow
a test-first workflow. Expected values should come from analytic calculations,
explicit small matrices, exact enumeration, manufactured numerical problems,
or neutral fixtures rather than from the implementation under test.

All contributions, if the project later opens to them, must be licensed under
Apache-2.0. Do not include credentials, private data, generated build output,
or local agent metadata in a change.
